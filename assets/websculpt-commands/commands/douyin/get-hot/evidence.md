# Evidence: douyin/get-hot

This document records the research and validation evidence for the `douyin/get-hot` command.

## Exploration Path

- Executed `websculpt command list` to check for existing Douyin commands.
- Consulted the browser automation guide for the CDP attach protocol.
- Attached to the user's Chrome via CDP to reuse the real login state and platform signatures.
- Navigated to `https://creator.douyin.com/creator-micro/creative-guidance`.
- Used DOM probing, snapshots, network request logging, and response-body inspection to understand the page.
- Interacted with UI elements (category tabs, sub-menu items, sort/time dropdowns) to trigger API requests and observe parameter changes.
- Key discovery: data comes from a JSON API rather than server-rendered HTML, so DOM parsing is not required.

## Verified URLs

- `https://creator.douyin.com/creator-micro/creative-guidance` — Douyin Creator Center creative insights page, reached via browser CDP.
- `https://creator.douyin.com/web/api/creator/material/center/billboard/?...` — Billboard data API, verified returning 200 with JSON response containing `{extra:{now:...}, item_list:[...], status_code:0}`.
- `https://creator.douyin.com/web/api/creator/material/center/config/?...` — Config API returning `{tag_list:[...], hot_word_tag_list:[...], hot_spot_tag_list:[...], ...}`, verified returning 200.

## Structural Evidence

### API Endpoint
`GET https://creator.douyin.com/web/api/creator/material/center/billboard/`

### Required Static Parameters
- `aid=2906` — app identifier, constant.
- `app_name=aweme_creator_platform` — app name, constant.
- `device_platform=web` — platform identifier, constant.
- `user_agent`, `referer`, `screen_width`, `screen_height`, `browser_language`, `browser_platform`, `browser_name`, `browser_version`, `browser_online`, `cookie_enabled`, `timezone_name` — populated from `navigator` / `screen` / `Intl` APIs.

### Required Session Parameters (dynamic, obtained from browser)
- `msToken` — session token, obtained from browser storage or page context.
- `a_bogus` — request signature, generated by the page's security SDK when available.
- `user_id` — logged-in user id, available from page state or config API.

### Functional Parameters
- `billboard_type` (integer): 1=Hot Videos, 2=Creation Hotspots, 3=Hot Topics, 4=Hot Challenges, 5=Hot Props, 6=Hot Music.
- `billboard_tag` (integer): Category code. 0=All. Mapping is fetched from the config API's `hot_word_tag_list`.
- `order_key` (integer, only for billboard_type=1): 1=Play count (default), 2=Likes, 3=Comments, 4=Hot score.
- `time_filter` (integer, only for billboard_type=1): 1=24 hours (default), 2=7 days, 3=30 days.
- `hot_search_type` (integer, only for billboard_type=2): Used instead of order_key/time_filter.

### Key Discovery: Signature Generation
The page loads a platform security SDK that exposes a signing function on `window`. The `a_bogus` parameter is generated by calling the SDK's signing method in the browser page context. The `msToken` can be extracted from browser storage or page state.

## Failure Signals

- **Anti-bot signature drift**: If the security SDK API changes or is removed, signing may fail. Check for console errors related to SDK loading.
- **Login state expired**: If `user_id` is missing or the billboard API returns empty/error, the user's Douyin creator session may have expired. User needs to re-login in Chrome.
- **API response structure change**: Watch for changes in `item_list` field names or non-zero `status_code` values.
- **Category/tag code changes**: The config API `tag_list` may add or modify codes. The command fetches the mapping dynamically from the config API.
- **billboard_type parameter changes**: Additional billboard types may be added or existing ones renumbered.
- **Network errors**: Timeout or 403/429 responses indicate rate limiting.

## Capture Assessment

This command should be captured. The exploration phase successfully:
- Identified the stable JSON API endpoint (not fragile DOM parsing).
- Mapped all functional parameters (billboard_type, billboard_tag, order_key, time_filter).
- Confirmed the data structure through response body inspection.
- Verified that a browser session is required for request signing and tokens.

The command provides high value: Douyin is a major short-video platform and its trending data is a critical signal for content trend analysis. No existing WebSculpt command covers this source.
